前面已经有了配置项的详细列表，但是这里还是要对velocity相关的配置项进行一下说明：


  • web.encoding

    • 配置网页的字符编码，默认配置为UTF-8


  • template.resource.loader.path

    • 配置模板文件的位置，默认为WEB-INF/views/


  • velocity.onfigurerloader.path

    • velocity的properties配置文件的路径，默认为空


  • template.load.charset

    • 模板文件的字符编码


  • system.devMode

    • 是否开启开发者模式，开发者模式下不缓存模板


  • velocity.velocimacro.library

    • velocity脚本文件的存储位置


  • velocity.user.directive

    • velocity用户自定义命令的位置


  • velocity.eventhandler.invalidreferences.class

    • Velocity处理无效参数的配置接口


  • velocity.eventhandler.referenceinsertion.class

    • velocity输出参数到页面前执行的接口，实现ReferenceInsertionEventHandler接口后配置在此处


  • velocity.eventhandler.nullset.class

    • Velocity处理空值的配置接口


  • velocity.layout.url

    • velocity的layout默认文件的位置，默认为 layout/default.vm


  • toolbox.xml.path

    • 配置vm-toolbox.xml的位置，默认为WEB-INF/conf/vm-toolbox.xml


  • web.vaild

    • 是否使用Web校验，默认为false


  • use.default.suffix.pattern

    • 使用默认的模版文件后缀，默认为true


  • melody.action.scan

    • controller的包扫描地址


  • velocity.in.jar.files

    • jar文件中的velocity模板文件位置


  • velocity.locale.country

    • 是否使用velocity模板的国际化配置,国家的配置，默认为true


  • velocity.locale.language

    • 是否使用velocity模板的国际化配置,语言，默认为true


  • velocity.locale.language.first

    • velocity模板的国际化配置,语言在前，默认为true
