action中返回参数的方法大体上分为四种 void、String、ModelandView、@ResponseBody，以下分别介绍。


void
  • 当方法返回值为void的时候，服务器返回的页面资源为url本身对应的页面文件资源，同时将ModelMap中的参数传递到页面
    • 点击查看示例
    String
  • 当方法返回值为String的时候，服务器会根据返回值提供页面资源，同时，返回方式分为forward和redirect两种。
    forward方式：服务器会直接将页面资源返回，使用方法是直接返回screen目录中的文件路径，需要注意的是结尾不能带；
    redirect方式：服务器会要求浏览器重定向到新的url，使用方法时返回的字符串以"redirect:"开头，并连接上完整链接或本地网页文件的path。
    • 点击查看示例

    ModelAndView
  • 方法的返回值也可以是ModelAndView，ModelAndView的对象可以设置返回的页面资源文件(forward和redirect均可)，同时也可以通过addObject方法设置返回页面的参数。
    • 点击查看示例
    @ResponseBody
  • 方法上注解了@ResponseBody之后，该方法的返回值就不会被spring处理，而是直接写入HttpResponse的body中，可以在页面上取到。
